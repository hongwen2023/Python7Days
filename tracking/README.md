# Claude Code 动态跟踪系统

这是一套让自己**持续跟上 Claude Code / agentic coding 最新进展**的个人系统。
核心思想：工具形态折旧极快，值得长期投资的是不变量（规格能力、验证闭环、任务切分、上下文经济学、循环设计）；
对变量部分，追求两个指标——**吸收延迟**（新能力发布 → 进入我的日常工作流隔了几天）和
**删除率**（每次大版本后删掉了多少过时的脚手架）。

## 循环设计（Loop Engineering）

> "I don't prompt Claude anymore. I have loops running that prompt Claude and figuring out
> what to do. My job is to write loops." — Boris Cherny, 2026-06

第五个不变量，2026-07-05 周报吸收：**高手的单位产出不是一次好的 prompt，而是一个设计良好的循环**
——`触发 → 执行 → 验证 → 反馈` 四环闭合、无需人在环中逐次驱动。

自查问题（每周/每月各问一次）：
- 本周我手动重复做了哪件事超过 2 次？它的触发条件、验证标准能不能写下来？能写下来就能变成循环。
- 我现有的循环里，哪个环最弱？（通常是"验证"——没有机器可判断的对错标准，循环就只是重复，不是进步。）

本系统自身就是示范：周报是"定时触发 → agent 扫描 → 对照台账验证 → 吸收/删除清单反馈"的循环。

## 文件地图

| 文件 | 作用 |
|---|---|
| `watchlist.md` | 关注人与信源清单（分三层），每日/每周扫描的对象 |
| `assumptions.md` | 我工作流中所有"因为工具做不到 X 所以我才做 Y"的假设台账 |
| `failure-cases.md` | 失败案例集，re-baseline 时用新模型/新版本重跑 |
| `weekly-report-prompt.md` | 每周自动周报的完整 prompt（定时任务和手动 `/weekly-scan` 共用） |
| `reports/` | 每周生成的周报归档 |

## 节奏

### 每日 ~10 分钟
只看 `watchlist.md` 第一、二层人物的 X/博客更新（用列表，不刷信息流）。
看到有价值的内容，随手记一行到当周周报草稿或直接更新 `assumptions.md`。

### 每周 ~45 分钟（自动 + 人工）
1. 定时任务自动生成周报到 `reports/`（也可手动触发：在 Claude Code 里运行 `/weekly-scan`）。
2. 人工只做两件事：
   - 从周报的"吸收清单"里挑 1 件事，本周实际用起来；
   - 从"删除清单"里挑 1 件事，删掉。

### 每月半天：re-baseline 仪式
1. 逐条过 `assumptions.md`，用当前模型/版本重新验证每条假设；
2. 从 `failure-cases.md` 挑 3-5 个历史失败案例，用原始 prompt 重跑，记录结果差异；
3. 删掉所有已失效假设对应的 workaround（配置、hook、脚本、习惯）；
4. 在 `assumptions.md` 底部记一行本次仪式的日期和删除数量。

## 维护原则

- 假设台账**只增改状态、不删行**——被推翻的假设标记为"已失效"留档，它们是你进步的刻度。
- 配置只增不减是工作流僵化的早期症状；每月删除数为 0 时要警惕。
- 周报建议 ≠ 必须执行；每周吸收 1 件、删除 1 件，细水长流。
