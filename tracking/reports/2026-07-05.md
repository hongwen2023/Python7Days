# Claude Code 周报 · 2026-07-05

> 窗口：2026-06-28 ~ 2026-07-05（首份周报，取最近 7 天）

## TL;DR

本周是大周：**Claude Sonnet 5 于 6 月 30 日发布并成为 Claude Code 默认模型，原生 1M 上下文窗口，8 月 31 日前 $2/$10 促销价**；Claude Code 近期版本把 **subagent 改为默认后台运行**（台账假设 #4 正式可销案）；社区完成了一次话语转向——从 "prompting" 到 **"loop engineering"**（Boris Cherny："我已经不再 prompt Claude 了，我的工作是写循环"）。

## 值得深读（3 篇）

1. **[Introducing Claude Sonnet 5（Anthropic 官方）](https://www.anthropic.com/news/claude-sonnet-5)** + [Caylent 的launch 分析](https://caylent.com/blog/claude-sonnet-5-launch-analysis-what-changed-what-matters-and-what-to-validate)——讲了什么：Sonnet 5 的能力边界（最"agentic"的 Sonnet，接近 Opus 4.8 的编码/工具使用水平）和定价窗口。对我意味着什么：**促销期是低成本做并行 agent 实验的两个月窗口**；Caylent 那篇提醒"先验证再切换"，正合台账复验的方法论。
2. **[Equipping agents for the real world with Agent Skills（Anthropic Engineering）](https://www.anthropic.com/engineering/equipping-agents-for-the-real-world-with-agent-skills)**——讲了什么：官方对 skills 作为 agent 能力封装单元的系统论述。对我意味着什么：skills 是当前官方主推的扩展机制，验证类 skill 仍应是第一优先级。
3. **["Loop Engineering: Stop Prompting, Start Designing"（TechTimes, 6/22）](https://www.techtimes.com/articles/318828/20260622/claude-code-loop-engineering-stop-prompting-start-designing-autonomous-agent-workflows.htm)**——讲了什么：6 月初 Steinberger 的爆帖 + [Boris 的名言](https://x.com/bcherny)（"I don't prompt Claude anymore... My job is to write loops"）+ Addy Osmani 的结构化文章，共同把"设计循环而非写提示"变成了正式词汇，且 Thariq 与 Sid Bidasaria 发布了 workflows 功能的实践指南。对我意味着什么：**这是评判标准的又一次位移**——上个月的关键词还是 context/harness engineering，本月是 loop engineering；我的周报系统本身就是一个 loop，方向对了。

## 吸收清单（最多 3 条）

1. **【分钟级】确认本地 Claude Code 已升级并默认使用 Sonnet 5**，趁 $2/$10 促销窗口把之前嫌贵没敢跑的 3-5 worktree 并行工作流跑起来，实测成本。
2. **【小时级】按 Simon Willison 7 月 3 日的建议做模型分层**：机械性实现工作交给低档模型跑 subagent，判断/评审/综合留在主循环（[simonwillison.net](https://simonwillison.net/)）。正好配合 subagent 默认后台化一起吸收。
3. **【分钟级】把 `/rewind`（可回到 `/clear` 之前）纳入日常**：换任务时可以更放心地 `/clear`，降低上下文管理的心智负担。

## 删除清单

- **台账 #4**（"subagent 会阻塞主会话 → 手动错峰安排子任务"）：v2.1.198 起 subagent 默认后台运行，workaround 正式作废，可在下次 re-baseline 时销案并删除相关习惯。
- **台账 #5**（"多 agent 编排需要自建脚本"）进一步弱化：workflows 功能已由 Thariq/Sid 发布实践指南，自建编排器的必要性再降一级。

## assumptions.md 待复验条目

- **#1（长上下文退化 → `/clear`/`/compact` 纪律）**：Sonnet 5 原生 1M 窗口后需要重测——窗口变大不等于没有 context rot，但退化阈值大概率显著后移；`/rewind` 也改变了 `/clear` 的成本。**这是本月 re-baseline 的头号实验。**
- **#5（自建编排）**：结合 workflows 实践指南重新评估"撞到产品边界"的判据。
- **#2（worktree 并行）**：暂无变化信号，但促销期适合做一次成本实测顺带复验。
