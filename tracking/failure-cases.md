# 失败案例集（Failure Cases）

> 每天挑一次失败的交互记三行；每月 re-baseline 时挑 3-5 个用原始 prompt 在新模型/新版本上重跑。
> 目的有二：① 找出失败的共性（通常 80% 集中在三类原因）；② 检测模型进步——曾经失败的案例开始成功，说明相关 workaround 可以删了。

## 记录模板

```
### FC-001 · 2026-07-05
- 任务：（一句话）
- 原始 prompt 要点：（或直接贴 prompt）
- 失败形态：（做错了什么 / 跑偏到哪）
- 根因分类：缺上下文 / 意图没说清 / 缺验证手段 / 模型能力不足 / 其他
- 缺的信息：（哪句话/哪个文件如果给了就不会错）
- 后续动作：（写进 CLAUDE.md？加 skill？加 hook？）

重跑记录：
| 日期 | 模型/版本 | 结果 | 结论 |
|---|---|---|---|
```
